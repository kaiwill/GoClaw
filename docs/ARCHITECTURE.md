# GoClaw 架构设计文档

## 1. 系统架构概述

GoClaw 是一个基于 Go 语言开发的 AI 助手框架，采用模块化设计，具有高性能、可扩展性和灵活性。系统由多个核心模块组成，通过明确的接口进行通信和协作。

## 2. 核心模块说明

### 2.1 代理引擎 (Agent)

代理引擎是 GoClaw 的核心组件，负责处理用户请求、执行工具、生成响应。

**主要功能：**
- 循环执行逻辑：接收消息 → 思考 → 执行工具 → 生成响应
- 上下文管理：维护对话历史和状态
- 工具调用：支持调用内置工具和自定义技能
- 记忆管理：与内存系统交互，存储和检索信息

**核心文件：**
- `pkg/agent/agent.go` - 代理核心定义
- `pkg/agent/loop.go` - 循环执行逻辑
- `pkg/agent/memory.go` - 内存交互

### 2.2 工具系统 (Tools)

工具系统提供了一系列内置工具，支持文件操作、网络请求、系统命令等功能。

**主要工具：**
- 文件操作：`file_read`, `file_write`, `file_edit`
- 系统命令：`shell`
- 网络请求：`http`, `web_fetch`, `web_search`
- Git 操作：`git_operations`
- 内存管理：`memory_store`, `memory_recall`, `memory_forget`
- 定时任务：`cron_add`, `cron_list`, `cron_remove`

**核心文件：**
- `pkg/tools/interface.go` - 工具接口定义
- `pkg/tools/file_tools.go` - 文件操作工具
- `pkg/tools/shell_tool.go` - 系统命令工具
- `pkg/tools/http_tools.go` - 网络请求工具

### 2.3 内存管理 (Memory)

内存管理系统负责存储和检索代理的记忆，支持多种存储后端。

**存储后端：**
- SQLite：本地持久化存储，支持 FTS5 全文搜索
- Qdrant：向量数据库，支持向量搜索
- None：内存存储，不持久化

**核心文件：**
- `pkg/memory/interface.go` - 内存接口定义
- `pkg/memory/sqlite.go` - SQLite 存储实现
- `pkg/memory/qdrant.go` - Qdrant 存储实现

### 2.4 消息通道 (Channels)

消息通道系统支持与多种消息平台集成，实现消息的发送和接收。

**支持的通道：**
- Telegram
- Discord
- Slack
- WhatsApp
- Matrix
- DingTalk
- Email

**核心文件：**
- `pkg/channels/interface.go` - 通道接口定义
- `pkg/channels/telegram.go` - Telegram 实现
- `pkg/channels/discord.go` - Discord 实现
- `pkg/channels/slack.go` - Slack 实现

### 2.5 LLM 提供商 (Providers)

LLM 提供商系统支持与多种大语言模型集成，实现模型的调用和管理。

**支持的提供商：**
- OpenAI
- Anthropic
- Gemini
- GLM
- Ollama
- Bedrock
- OpenRouter
- Bailian (阿里云百炼)

**核心文件：**
- `pkg/providers/interface.go` - 提供商接口定义
- `pkg/providers/openai.go` - OpenAI 实现
- `pkg/providers/anthropic.go` - Anthropic 实现
- `pkg/providers/gemini.go` - Gemini 实现

### 2.6 API 网关 (Gateway)

API 网关提供了 HTTP API、WebSocket 和 SSE 流等接口，支持与外部系统集成。

**主要接口：**
- RESTful API：用于发送消息和管理系统
- WebSocket：用于实时消息推送
- SSE 流：用于流式响应
- OpenAI 兼容接口：兼容 OpenAI API 格式

**核心文件：**
- `pkg/gateway/server.go` - HTTP 服务器
- `pkg/gateway/websocket.go` - WebSocket 实现

## 5. 配置管理

GoClaw 使用 TOML 格式的配置文件进行配置管理，配置文件位于 `~/.goclaw/config.toml`。

### 5.1 主要配置项

- **provider**：LLM 提供商配置
- **gateway**：API 网关配置
- **memory**：内存管理配置
- **channels**：消息通道配置
- **security**：安全策略配置

### 5.2 环境变量

GoClaw 支持通过环境变量覆盖配置文件中的设置，环境变量前缀为 `GOCLAW_`。

## 6. 部署架构

### 6.1 单节点部署

适合开发和测试环境：
- 所有组件在同一台机器上运行
- 使用 SQLite 作为内存存储
- 直接访问 LLM API

### 6.2 分布式部署

适合生产环境：
- 代理引擎和 API 网关分离部署
- 使用 Qdrant 作为分布式向量存储
- 通过负载均衡器分发请求

## 7. 扩展性设计

GoClaw 采用插件式架构，支持以下扩展方式：

1. **自定义工具**：通过实现 `Tool` 接口添加新工具
2. **自定义技能**：通过技能系统添加新技能
3. **自定义通道**：通过实现 `Channel` 接口添加新消息通道
4. **自定义提供商**：通过实现 `Provider` 接口添加新 LLM 提供商
5. **自定义内存后端**：通过实现 `MemoryBackend` 接口添加新存储后端

## 8. 安全考虑

### 8.1 安全策略

- **工具权限控制**：限制工具的执行权限
- **API 认证**：保护 API 接口
- **数据加密**：敏感数据加密存储
- **输入验证**：验证用户输入

### 8.2 风险缓解

- **沙箱隔离**：隔离工具执行环境
- **速率限制**：防止 API 滥用
- **日志审计**：记录系统操作
- **错误处理**：安全处理错误信息

## 9. 性能优化

### 9.1 优化策略

- **缓存机制**：缓存频繁访问的数据
- **异步处理**：使用 goroutines 处理并发请求
- **连接池**：复用数据库连接
- **批处理**：批量处理请求

### 9.2 监控指标

- **响应时间**：API 响应时间
- **内存使用**：系统内存使用情况
- **CPU 使用率**：系统 CPU 使用情况
- **请求量**：API 请求量
